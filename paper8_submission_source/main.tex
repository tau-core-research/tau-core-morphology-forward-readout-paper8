\begin{abstract}
Paper 1 found that residual-blind external structural disturbance labels are associated with increased low-acceleration residual scatter in SPARC galaxies. Paper 2 reversed the question and showed that fixed residual-shape features can infer those disturbance labels better than shuffled controls, while remaining non-unique with respect to MOND-simple and empirical RAR-like baselines. Paper 3 then built a candidate/control framework and introduced an endpoint-conditioned required-$S_\tau$ diagnostic: what preparation/readout factor would be needed to absorb a measured residual? This Paper 8 proposal turns that inverse diagnostic into a forward validation gate. The proposed gate predeclares a residual-blind observed morphology handle $K_{\rm obs}$, audits whether it should be used directly or mapped to a readout-relevant proxy $K_{\rm readout}$, maps $K_{\rm readout}$ to a Tau Core morphology-conditioned readout shell, computes a forward $\delta g^K$ without using required $S_\tau$ as a predictor, and compares matched-family residual structure against wrong-family, shuffled-label, Newtonian baryonic, MOND-simple, and empirical RAR controls. The package also reaches a concrete NGC4013 milestone: a residual-blind source rule and formula-freeze gate now define an approximate mixed 4D readout protocol, \path{MIXED\_FORMULA\_FREEZE\_READY\_NOT\_RETROACTIVE\_ENDPOINT}, without converting the existing diagnostic score into retrospective validation. The permitted claim is methodological: the morphology-matched forward-readout gate is a falsifiable, Tau-Core-specific next endpoint for the Paper 1--3 program. This paper does not claim that Tau Core has been empirically validated, that a universal galaxy law has been derived, or that the matched-family endpoint has already beaten MOND/RAR on real SPARC data.
\end{abstract}

\section{Purpose and claim boundary}

The first three SPARC residual papers define a cautious progression. Paper 1 asks whether external structural disturbance is associated with residual scatter \cite{Paper1Package}. Paper 2 asks whether residual-shape features can recover external disturbance labels better than shuffled labels \cite{Paper2Package}. Paper 3 turns the association into a candidate/control framework and asks what $S_\tau$ would be required to absorb the observed residual \cite{Paper3Package}.

That Paper 3 $S_\tau$ diagnostic is useful but partly inverse. It asks:
\[
\text{given the measured endpoint, what } S_\tau(R) \text{ would absorb it?}
\]
The present paper defines the next gate:
\[
\begin{aligned}
&\text{residual-blind observed morphology handle }K_{\rm obs}\\
&\quad\longrightarrow\text{ readout-relevant proxy }K_{\rm readout}\\
&\quad\longrightarrow\text{ forward readout }\delta g^K
\text{ before endpoint scoring.}
\end{aligned}
\]

The main claim is deliberately narrow:
\begin{quote}
The Tau Core gravity bridge can sharpen Papers 1--3 by replacing inverse required-$S_\tau$ residual absorption with a morphology-matched forward-readout candidate/control gate.
\end{quote}

This paper does not claim:
\begin{itemize}[leftmargin=*]
\item that Tau Core is proven;
\item that the morphology-matched gate has already won on real SPARC endpoints;
\item that MOND, RAR, or Newtonian baryonic baselines have been superseded;
\item that non-axisymmetric morphology classes can be fully tested from one-dimensional rotation curves alone;
\item that the morphology families below are the final physical taxonomy.
\end{itemize}

\section{From Paper 3 inverse diagnostics to forward readout}

Paper 3 used an endpoint-conditioned diagnostic of the form
\[
S_{\tau,\mathrm{req}}(R)=
\frac{V_{\rm obs}(R)/V_N(R)-1}{\alpha\ln(1+a_0/a_N(R))}.
\]
This quantity is descriptive. Because it uses $V_{\rm obs}$, it cannot be treated as an independent predictor. Its correct role is to identify what kind of future nonleaky rule would be required.

The forward replacement is:
\[
K_{\rm obs} \longrightarrow K_{\rm readout}
\longrightarrow \mathcal{F}_{K_{\rm readout}}
\longrightarrow \delta g^{K_{\rm readout}}(R)
\longrightarrow \mathrm{Score}(g_{\rm Newt}+\delta g^{K_{\rm readout}}).
\]
Here $K_{\rm obs}$ is the residual-blind observed 4D morphology handle, $K_{\rm readout}$ is the predeclared readout-relevant morphology proxy, $\mathcal{F}_{K_{\rm readout}}$ is the selected readout family, and $\delta g^{K_{\rm readout}}$ is computed without using $S_{\tau,\mathrm{req}}$, endpoint residual gain, or post-hoc family selection. The equality $K_{\rm obs}=K_{\rm readout}$ is allowed but not assumed; this is a Tau Core bridge consequence of treating morphology as a projection/readout structure rather than only a present-day visual class.

\section{MORPHOLOGY-MATCHED-FORWARD-READOUT-GATE}

For a galaxy $g$, let
\[
K_{{\rm obs},g}
\]
be its residual-blind observed 4D morphology handle, and let
\[
K_{{\rm readout},g}\in\mathcal{K}
\]
be the predeclared readout-relevant morphology proxy used for formula-shell selection. Let $\mathcal{F}_K$ denote the Tau Core readout shell assigned to readout proxy class $K$. Let
\[
R_{g,K}
\]
be the residual score after applying the forward readout family $\mathcal{F}_K$ to galaxy $g$ under a frozen geometry and amplitude policy.

The matched-family score is
\[
R_{g,\mathrm{matched}}:=R_{g,K_{{\rm readout},g}}.
\]
The wrong-family score is
\[
R_{g,\mathrm{wrong}}:=
\frac{1}{|\mathcal{K}|-1}
\sum_{K\neq K_{{\rm readout},g}}R_{g,K}.
\]
The central paper-level endpoint can be written as
\[
\Delta_{\rm matched}
=
\mathrm{RMS}_{g}\!\left(R_{g,\mathrm{wrong}}\right)
-
\mathrm{RMS}_{g}\!\left(R_{g,\mathrm{matched}}\right).
\]
An equivalent rank endpoint is
\[
\mathrm{rank}_{g}(K_{{\rm readout},g})
=
\mathrm{rank}\bigl(R_{g,K_{{\rm readout},g}}\;\mathrm{among}\;\{R_{g,K}:K\in\mathcal{K}\}\bigr).
\]

The gate passes only if the predeclared matched family improves residual structure more than:
\begin{enumerate}[leftmargin=*]
\item wrong morphology families;
\item shuffled morphology labels;
\item Newtonian baryonic baseline;
\item MOND-simple baseline;
\item empirical RAR-like baseline.
\end{enumerate}

This is Tau-Core-specific in a way that ordinary MOND/RAR comparisons are not: the discriminating claim is not merely lower residuals, but morphology-indexed readout specificity.

\section{Available-data Tau-proxy preflight}

The repository also includes a first available-data preflight of the matched-family idea. This is not the final Paper 8 endpoint, because the family functions are still proxy functions built from the locally available Tau-proxy channel rather than the final morphology-specific $\delta g^K$ readout shells. The preflight nevertheless tests the important next question: whether a metadata-proxy morphology family assigned before scoring performs better than wrong Tau-proxy families on held-out galaxies.

In the current 175-galaxy point-level packet, family amplitudes are fit on the train split only. On the 44-galaxy holdout split, the morphology-matched proxy beats the wrong-family mean in 0.568 of galaxies, beats the TPG/v6 comparator in 0.636 of galaxies, and beats the MOND comparator in 0.614 of galaxies. A deterministic 1000-permutation shuffled-label null gives a holdout null mean of 0.0133 for the mean matched-minus-wrong endpoint, compared with the observed value $-0.0326$, with $p\simeq0.264$ for shuffled labels being at least as good. Thus the available proxy signal is positive but not yet a discovery-level morphology-specific result. These numbers are recorded as a preparation result, not as empirical validation of Tau Core. They motivate replacing the generic Tau proxy with source-native morphology-family readout shells and a stronger shuffled-label null in the final endpoint.

A second preflight replaces the single available Tau-proxy channel with different radial formula-shell proxies assigned by morphology class. This is closer in form to the intended Paper 8 family test, but the current available-data version is weaker: on the same 44-galaxy holdout split it beats the wrong-shell mean in 0.500 of galaxies, beats TPG/v6 in 0.477 of galaxies, and has a shuffled-null $p\simeq0.479$ for the mean matched-minus-wrong endpoint. This negative preflight result is useful. It indicates that merely assigning naive radial shell shapes is not sufficient; the final endpoint needs source-native morphology labels, dimensionally audited 4D readout shells, and a nonleaky amplitude policy.

The source-native bridge formula preflight then consumes a declared morphology-parameter manifest and evaluates the concrete morphology formulas as $\delta v_K^2$ readout kernels: the scale-tail $n=2$ branch, the finite exponential disk Freeman/Bessel kernel, the compact finite-source exterior response, and the thick/flared damped vertical-kernel response. With the same available-data morphology labels and train-only amplitudes, the 44-galaxy holdout split gives a much stronger morphology-specific endpoint: the matched bridge formula beats the wrong-formula mean in 0.886 of galaxies, with a 1000-permutation shuffled-label null giving $p\simeq0.002$ for the mean matched-minus-wrong endpoint. However, it beats TPG/v6 and MOND in only 0.477 of holdout galaxies each, and its mean matched-minus-baseline residual is still worse than both baselines. Thus the bridge-formula result supports the Paper 8 morphology-family specificity hypothesis, but it is not an empirical claim that Tau Core outperforms TPG/MOND.

A readout-mixture proxy diagnostic then tests the natural bridge refinement that a galaxy may carry a mixture of readout components rather than a single hard family. This diagnostic keeps the same concrete bridge kernels and uses residual-blind available-data proxy weights. The current holdout result is negative: the mixture beats the hard matched family in only 0.455 of galaxies, beats TPG/v6 in 0.500, and beats MOND in 0.455. This is a useful failure map, not a defeat of the bridge idea. It shows that a mixture layer is only meaningful if its weights are supplied by accepted morphology-memory/readout observables, not by coarse present-day 4D proxy heuristics.

The package therefore adds a readout-state vector intake gate. This gate does not score rotations. It lists the residual-blind source observables required before the mixture weights can be promoted into an accepted Tau-side readout-state vector: disk scale/decomposition, outer-disk or HI tail support, compact-core support, vertical flare/projection evidence, morphology-memory/history evidence, and a source-normalization rule. Under the strict accepted-input rule, the current package remains blocked: the memory/history layer is proxy-only and the Tau-side normalization rule is now present only as a theory-conditional candidate, not as an accepted physical normalization law.

A confidence diagnostic on the manifest shows why the baseline claim remains blocked. Restricting the holdout set to confidence $\geq0.75$ preserves the matched-vs-wrong signal, with matched formulas beating wrong formulas in 0.906 of galaxies, but the TPG/MOND comparison is still mixed. Removing low-inclination or large-distance-error objects improves some baseline comparisons, so data quality matters. It is not the whole bottleneck: the remaining blockers are source-native morphology-scale extraction, amplitude policy, and normalization of the bridge formula kernels.

A baseline-success morphology audit then asks the complementary control question: where TPG/MOND/Newton/RAR already work, what Tau Core morphology/readout regime might that indicate? In the current holdout split, TPG/v6 wins are concentrated in scale-tail/irregular rows and have zero current-proxy/readout-memory agreement, suggesting a possible smooth closure-like or memory-integrated readout regime. Tau-matched wins have higher current-readout consistency, while MOND wins are compatible with simpler radial low-acceleration or diffuse-disk scaling regimes. Where Newtonian wins in the smaller conventional tables, the claim-safe Tau Core interpretation is a quiet or regular baryonic-readout regime, not a refutation of morphology specificity elsewhere. Thus baseline success is not discarded. Newtonian-good rows become quiet baryonic-readout controls; MOND/RAR-good rows become scalarized radial-readout controls; TPG-good rows become closure-like or memory-integrated readout controls; and Tau-family-good rows become cases where the present morphology proxy may be closer to $K_{\rm readout}$. This makes the final question sharper: whether baseline success zones and Tau-family success zones map to predeclared readout regimes rather than to post-hoc model choice.

The same audit gives a conditional limiting-regime interpretation of the comparison baselines. In a quiet, current-regular baryonic-readout state with small $\alpha_\tau$ and suppressed quotient-visible morphology residual, the Tau Core readout should reduce to the Newtonian weak-field limit, $g_{\rm eff}\to g_{\rm Newt}$. In a scalarized radial low-acceleration state, where morphology dependence is weak or averaged and the needed scale is supplied by Tau-side closure/readout data rather than inserted as a fit law, a MOND/RAR-like solved response can be a special effective limit. In a smooth closure-like or memory-integrated state with an outer logarithmic or near-flat-speed solved branch, a TPG-like formula can be the natural effective readout. If the same residual descends through a metric-compatible and gauge-safe weak-field readout map, the branch is RMOND-facing, meaning suitable for relativistic/metric audit, not yet a derived relativistic MOND theory. This is the intended bridge statement: successful Newtonian, MOND/RAR, TPG, and RMOND-facing behavior should be explained as conditional Tau Core readout limits when their regime assumptions hold, while the endpoint still tests whether these regimes were predeclared rather than selected after seeing residuals.

The bridge status of these limiting forms is deliberately uneven. The Newtonian limit is formula-derived when $\delta g_{\rm morph}\to0$. The TPG-like shape is formula-derived under the $n=2$ source-tail gate,
\[
\sigma_{\rm morph}\sim {A\over r^2}
\quad\Longrightarrow\quad
\delta g\sim {A\over r}
\quad\Longrightarrow\quad
\delta\Phi\sim A\log r .
\]
The fixed historical TPG branch remains formula-conditional because the finite-load constants, branch choice, and historical normalization require additional Tau-side closure/readout input. The MOND/RAR-like scaling is also formula-conditional:
\[
A^2 = G M_b g_*
\quad\Longrightarrow\quad
\delta g=\sqrt{g_N g_*}.
\]
Here $g_*:=\lambda_*\ell_*$ is an internal Tau-side acceleration-scale object only if $\lambda_*$ is a residual-blind closure/readout stiffness with dimension $T^{-2}$ and $\ell_*$ is an admissible readout length. This is not a numerical $a_0$ validation, not a universal MOND-scale derivation, and not an empirical baseline-superiority claim. The RMOND-facing branch remains readout-form-only until a covariant field equation, conservation audit, and lensing sector are supplied.

This gives a conditional baseline-selection theorem. Let $R_g$ be a residual-blind readout-regime label assigned to a galaxy before rotation endpoint scoring. If $R_g$ satisfies the quiet baryonic-readout assumptions, then the suppressed-residual limit gives the Newtonian branch. If $R_g$ satisfies the scalarized radial low-acceleration assumptions and an internal $g_*=\lambda_*\ell_*$ scale has been supplied, then the solved response has a MOND/RAR-like branch. If $R_g$ satisfies the smooth closure/readout source-tail assumptions, in particular the $n=2$ tail $\sigma_{\rm morph}\sim A/r^2$, then the solved response has a TPG-like logarithmic or near-flat-speed branch. If $R_g$ satisfies the gauge-safe metric-descent assumptions, then the branch is RMOND-facing as a weak-field readout form. The proof is by substitution of the corresponding regime assumptions into the bridge formula-status ladder above. The theorem does not choose a baseline after seeing residuals: if several regime predicates hold, the galaxy is an overlap/control case; if none holds, the current bridge has not explained that baseline success.

The resulting empirical target is a morphology-information-gain test rather than a full high-dimensional morphology fit. A complete galaxy morphology would require many source variables, including disk scale, compact support, bar strength, ring structure, thickness or flare, HI tails, projection, environment, and morphology-memory indicators. The claim-safe test is lower-dimensional: as residual-blind morphology/readout information is refined from a coarse $K_{\rm obs}$ label, to source-reviewed $K_{\rm readout}$, to a readout-state vector, to source-native scales and amplitudes, the forward Tau Core readout should improve in predeclared scores. A pass is not required to be monotonic for every galaxy, but the population-level trend should improve matched-vs-wrong rank, shuffled-label separation, residual RMS, baseline competitiveness, or readout-regime classification. A fail occurs if additional morphology information gives no improvement, or if improvement appears only when rotation residuals are used to choose labels, weights, scales, or gates.

The repository now includes a first executable version of this test. It assembles four currently available levels: $L0$ coarse formula-shell proxy, $L1$ source-native bridge formula shell, $L2$ readout-state mixture proxy, and $L3$ source-native shells with train-selected normalization. On the current holdout split, the $L0\to L1$ transition strengthens family specificity but does not improve raw RMSE; $L1\to L2$ is mixed or negative because the mixture weights are still proxy-only; and $L2\to L3$ improves the proxy mixture layer. The associated full-sample source expansion records 175 SPARC rows, 75 S4G scale-radius candidates, 31 DustPedia source-candidate matches, 171 SPARC HI-ready galaxies, two PHANGS public-sample matches, 172 L2 tail candidates, 48 L2 compact candidates, 19 L2 bar candidates, zero MUSE-ready velocity-field matches, and no endpoint-ready accepted readout-state vector components. A separate L2 weight-intake preflight converts these source candidates into residual-blind candidate readout weights for 174 of 175 galaxies, with one uninformative fallback; nonzero candidate support appears for 172 tail, 75 exponential-disk, 48 compact, and 107 thick/flared channels. These weights are not accepted Tau-side readout-state weights and are not endpoint scores. When scored as an endpoint stress test, the L2 intake layer remains mixed/negative on holdout: it beats the old L2 mixture proxy in 0.409 of galaxies, beats the hard source-native family in 0.409, beats TPG/v6 in 0.477, beats MOND in 0.432, and has median intake-minus-old-L2 RMSE $+0.847$.

The package then introduces a theory-conditional residual-blind Tau-side source-normalization candidate,
\[
\widehat K_{gK}(r)={K_{gK}(r)\over {\rm median}_r |K_{gK}(r)|},
\qquad
c_g={{\rm median}_r \max(v_{\rm v6}^2-v_n^2,0)\over {\rm median}_r v_{\rm v6}^2},
\]
\[
\delta v_{gK}^2(r)=
\sigma_K\,e_{gK}\,w_{gK}\,c_g\,{\rm median}_r(v_n^2)\,\widehat K_{gK}(r).
\]
Here $w_{gK}$ is the residual-blind L2 intake weight, $e_{gK}$ is the residual-blind source-evidence gate, and $\sigma_K$ is a predeclared readout-orientation sign: compact and tail components are positive, while exponential-disk and thick/flared components are negative. A separate derivation manifest records these signs and gates before endpoint scoring and audits the rule as dimensionally consistent for a $\delta v^2$ candidate; the manifest marks the orientation signs as theory-conditional bridge derivations and the proxy attenuation as the coarse executable representative of the conservative Tau-side readout-admission product, with standard proxy value $0.70\times0.70\times0.85\times0.85=0.354025$. No observed velocity endpoint, rotation residual, required-$S_\tau$ diagnostic, or best-family choice enters this normalization. On holdout, this source-normalized L2 preflight beats the old L2 intake endpoint in 0.568 of galaxies, beats TPG/v6 in 0.455, beats MOND in 0.545, and has median source-normalized-minus-old-L2 RMSE $-0.272$. This is an improvement over the raw L2 intake endpoint, but not an accepted normalization law. A post-$E_\tau$ freeze-readiness audit still blocks endpoint use for all 175 galaxies, but the blocker has changed: the proxy-gate blocker is resolved for all 175 rows by the derived coarse-grid $E_\tau$ product, missing Tau-side normalization blocks 0 rows, and a source-native orientation-promotion gate now promotes three of four family orientations. In galaxy-level active-component terms, 67 of 175 rows are orientation-ready and 108 remain orientation-blocked. A stricter memory/projection gate then splits those 67 rows into 47 projection-blocked rows, 19 memory/history-blocked rows, and one memory/projection-ready candidate that still lacks accepted per-galaxy $q_i$ and normalization-law acceptance. Dominant components have source-candidate support in 141 rows, are proxy/partial-only in 31 rows, and lack dominant source support in three rows. The preserved boundary is therefore sharper: source coverage plus a residual-blind normalization candidate is not yet enough; the remaining thick/flared orientation rows, projection caveats, memory/history acceptance, accepted per-galaxy evidence assignments, and an accepted normalization law are still required before endpoint freeze. This is a failure map and data-acquisition guide, not a monotonicity proof.

The repository also records a lane-expansion audit so that the sample can grow without weakening the accepted-lane claim. Under the current residual-blind gates, only one galaxy is a strict memory/projection-ready candidate, but 66 additional galaxies are caution/proxy-supported because they are orientation-ready while still carrying projection or memory/history caveats. Thus 67 of 175 galaxies are analysis-includable in strict-plus-caution support lanes, while 108 remain acquisition-required. The caution lane is explicitly not accepted evidence; it is a support/sensitivity lane for deciding which projection, vertical-structure, and memory/history source data should be acquired next.

When the existing endpoint diagnostics are sliced by these lanes, the current holdout strict-plus-caution subset contains 16 galaxies. On that subset, the source-native hard-family formula keeps a morphology-specific signal, beating the wrong-family mean in 0.8125 of galaxies, but it is only mixed against TPG/v6 and MOND, with win fractions 0.5000 and 0.4375. The Tau-side evidence-measure L2 normalization is weaker on the same lane, beating TPG/v6 in 0.3750 and MOND in 0.3125 of galaxies. This is not a validation failure and not a discovery claim; it is a preparation diagnostic showing that the expanded lane is useful for sensitivity and acquisition planning while baseline-superiority remains blocked.

Splitting the caution lane by allowed use sharpens the failure map. In the holdout projection-caveat sub-lane, the source-native hard-family formula still beats the wrong-family mean in 0.7857 of galaxies, while the Tau-side evidence-measure L2 normalization beats TPG/v6 in only 0.3571 and MOND in only 0.2857. Thus the present weakness is not primarily the disappearance of morphology-family specificity; it is the projection/scale and source-normalization layer needed to turn that specificity into a competitive solved response.

Two repair audits make this more operational. The projection/scale repair map finds 71 of 175 rows with no projection/scale repair required, 34 requiring vertical-geometry source evidence, 26 requiring inclination/projection review, 30 requiring new distance/scale source support, and 14 that are plausibly repairable with existing scale-source information plus a distance/scale audit. A source-normalization failure-mode audit then shows that, inside the holdout projection-caveat sub-lane, 7 of 14 rows are projection-scale normalization failures: the hard-family morphology-specific signal is present, but the Tau evidence L2 normalization fails to transfer it into a baseline-competitive solved response. This identifies a concrete repair target without changing any endpoint gate.

The bridge-level promotion target is not to defend a universal proxy constant. It is to replace the coarse proxy bin by a residual-blind evidence measure $e_{gK}=E_\tau(g,K)$ built from source presence, geometry relevance, projection safety, morphology-memory reliability, and resolution adequacy. In the current Tau Core bridge derivation, accepted/proxy/missing are treated as coarse readout-admission states, proxy is the minimal admitted nonmissing state, and conservative three-status mass normalization gives $E_{\rm proxy}^\ast=1/3$. Combined with the standard proxy template this gives $\epsilon_\ast=0.157399\ldots$, whose coarse-grid representatives are strong proxy $0.85$ and ordinary proxy $0.70$. The resulting standard proxy product is $0.354025$. This is stronger than a hand-fixed $0.35$ protocol constant, while still not a claim that the conservative three-status evidence geometry is the unique final Tau-side evidence law.

The repository now includes a first executable version of this promotion target. For all nonzero L2 components, it builds
\[
E_\tau=q_{\rm source}q_{\rm geometry}q_{\rm projection}q_{\rm memory}q_{\rm resolution}.
\]
On the full sample, this audits 406 components, including 105 proxy/partial components. The median proxy value is $E_\tau=0.354025$, with median proxy-minus-fixed-$0.35$ equal to $+0.004025$. Replacing the fixed proxy-bin gate by this $E_\tau(g,K)$ candidate leaves the holdout win fractions unchanged relative to the fixed-gate source-normalized preflight: it beats the old L2 intake endpoint in 0.568 of galaxies, TPG/v6 in 0.455, and MOND in 0.545, while the median $E_\tau$-minus-old-L2 RMSE is $-0.236$. This is not model selection. It shows that the fixed proxy gate can be promoted to a residual-blind evidence-measure candidate without using endpoint residuals. The proxy ladder is now derived inside the current conservative readout-admission geometry; the remaining caveat is that the galaxy/component $q_i$ assignments are still theory-candidate source/readout assignments rather than accepted source-native observables.

The executable audit also records per-factor derivation status. Accepted and missing source gates are definition-derived limit cases of the evidence measure. Proxy/partial rows are kept in a separate theory-candidate factor-geometry status until their source, geometry, projection, memory, and resolution factors are fixed by residual-blind Tau-side readout rules rather than endpoint performance.

A sensitivity audit varies only the conditional orientation signs and source-evidence gates. It is not a model-selection step. On holdout, the primary proxy-gate value $e_{\rm proxy}=0.35$ beats the old L2 intake endpoint in 0.568 of galaxies. Removing proxy support gives 0.545. Raising proxy support to unity gives 0.523 against the old L2 intake endpoint and 0.523 against TPG/v6, but this stronger TPG-facing number cannot be used as a selected endpoint because the gate is fixed by the conservative readout-admission derivation rather than by endpoint performance. All-positive orientation signs reduce the old-L2 win fraction to 0.477, and all-negative signs reduce it to 0.432. Thus the orientation structure is not cosmetic; after the proxy-gate derivation, the source-native orientation gate partially promotes the signs, and the remaining freeze blockers are the thick/flared orientation source layer plus accepted per-galaxy evidence and morphology-memory/projection assignments.

An amplitude-policy diagnostic supports the same conclusion. Keeping the source-native bridge kernels fixed, the family-unconstrained policy gives the strongest matched-vs-wrong signal but remains weak against TPG/MOND baselines. A simple family-to-global shrinkage policy preserves most of the matched-vs-wrong signal, 0.864 on holdout, while raising the MOND win fraction to 0.636 and reducing the mean MOND gap. This is not a selected endpoint claim; it identifies amplitude normalization as a required Tau-side modeling gate before any baseline-superiority claim is allowed.

A shrinkage-path scan makes this more precise. Varying the family amplitude weight from 0 to 1 shows that the current best compromise lies near family weight 0.4--0.5. At weight 0.5, the holdout matched-vs-wrong fraction is 0.864 and the MOND win fraction is 0.636, while TPG/v6 remains at 0.477. This identifies the normalization target for the next Tau-side gate: a physical source-normalization rule should recover a partially pooled family amplitude, not a fully free family amplitude chosen only for endpoint fit.

A train-selected shrinkage diagnostic reduces the post-hoc risk in that conclusion. Four selection rules using train split metrics only all select family weight 0.40. Evaluated on the 44-galaxy holdout split, this train-selected policy keeps matched-vs-wrong specificity at 0.818, beats MOND in 0.614 of galaxies, and remains mixed against TPG/v6 at 0.477. This supports a stable partially pooled normalization range, but it is still not a validated amplitude-normalization law or a baseline-superiority endpoint.

A family-breakdown diagnostic then asks where this signal lives. Under the train-selected 0.40 shrinkage policy, the holdout matched-vs-wrong specificity is strongest for the scale-tail spiral and thick/flared families, both at 0.900. The thick/flared row is also competitive against both TPG/v6 and MOND in this diagnostic, while the scale-tail row remains MOND-competitive but TPG-blocked. The exponential-disk and compact finite-source rows are weaker under the current available-data manifest. This is useful as a failure map: it points to family-specific source normalization and morphology-observable extraction as the next modeling gates, not to a validated family-level empirical claim.

An observable-quality diagnostic separates part of that failure map. Joining the train-selected scores to the residual-blind morphology manifest shows that, on holdout, the thick/flared family is the cleanest current-best-case row, while the compact finite-source, exponential-disk, and scale-tail rows are quality-limited under the available-data proxies. In the clean holdout manifest subset, matched formulas beat wrong formulas in 0.900 of galaxies and beat MOND in 0.750, whereas quality-caveated rows fall to 0.750 and 0.500. This does not rescue the endpoint by filtering after the fact; it identifies residual-blind morphology-observable extraction as a required predeclared gate for the next run.

A predeclared quality-gate diagnostic makes that next run more concrete. Under the fixed train-selected shrinkage policy, the clean-manifest gate and the confidence-$\geq0.75$-and-clean gate each retain 20 holdout galaxies across all four formula families, with matched-vs-wrong specificity 0.900, TPG/v6 win fraction 0.500, and MOND win fraction 0.750. A broader no-large-distance-error gate retains 26 holdout galaxies and remains candidate-level, with matched-vs-wrong 0.885, TPG/v6 0.538, and MOND 0.731. These are not selected discovery endpoints; they are candidate observability rules that must be declared before endpoint scoring in a future Paper 8 run.

The corresponding shuffled-label null reveals the tradeoff. On holdout, the full all-galaxy gate has the strongest shuffled-null support for morphology specificity, with $p\simeq0.036$ for the beats-wrong fraction and $p\simeq0.013$ for the mean matched-minus-wrong endpoint. The no-low-inclination gate gives similar values, $p\simeq0.041$ and $p\simeq0.011$. Cleaner gates improve the MOND/TPG comparison but have weaker shuffled-null support because the sample shrinks. Thus Paper 8 should not silently choose the cleanest-looking subset; it should predeclare whether the endpoint prioritizes observability-clean baseline competitiveness or larger-sample morphology-label specificity.

Combining these diagnostics gives a concrete preparation-state endpoint decision matrix. The current natural primary endpoint candidate is the no-low-inclination gate: it retains 35 holdout galaxies, remains baseline-competitive, and has strong shuffled-null support. The full all-galaxy gate should remain a fuller-sample morphology-specificity support lane, while the no-large-distance-error gate is the strongest secondary baseline-competitiveness lane. These roles must be predeclared before any future endpoint scoring; they are not retrospective proof of Tau Core.

The repository therefore records a predeclared endpoint protocol sheet. It freezes the current preparation-state choices: primary lane no-low-inclination, fuller-sample support lane all, secondary baseline lane no-large-distance-error, train-selected family-to-global shrinkage with weight 0.40, residual-blind morphology-family assignment, matched-vs-wrong plus shuffled-family null specificity metrics, and TPG/v6 plus MOND comparator metrics. It also records forbidden inputs, including endpoint residual gain, required-$S_\tau$, post-hoc gate choice, and per-galaxy amplitude tuning. This is the guardrail for the next run, not a claim that the present diagnostic run validates Tau Core.

\section{Minimal numerical protocol}

The minimal public endpoint protocol is:
\begin{enumerate}[leftmargin=*]
\item Assign an external or residual-blind observed 4D morphology handle $K_{{\rm obs},g}$ to each galaxy.
\item Predeclare whether $K_{{\rm obs},g}$ is used directly or mapped to a readout-relevant proxy $K_{{\rm readout},g}$ using only residual-blind source evidence.
\item Select a formula shell $\mathcal{F}_{K_{{\rm readout},g}}$ from the predeclared registry.
\item Fix geometry parameters from morphology, not from residual endpoints.
\item Use either a frozen amplitude rule or a tightly bounded amplitude fit.
\item Compute the matched-family score.
\item Compute wrong-family scores for all other candidate families.
\item Run shuffled-$K$ nulls.
\item Compare against Newtonian baryonic, MOND-simple, and empirical RAR baselines.
\end{enumerate}

The information-gain version of the protocol repeats this endpoint at
predeclared morphology-information levels:
\[
\begin{aligned}
L0 &: K_{\rm obs},\\
L1 &: K_{\rm readout},\\
L2 &: {\bf q}_{\rm readout},\\
L3 &: \hbox{source-native scales and normalization},\\
L4 &: \hbox{velocity-field / HI / history-enriched data}.
\end{aligned}
\]
The target is not a per-galaxy free morphology fit. The target is a
population-level improvement curve under residual-blind information upgrades.

For SPARC-only first-pass work, the cleanest initial family set is:
\[
\{K_{\rm tail},K_{\rm exp},K_{\rm compact},K_{\rm ring},K_{\rm flare}\}.
\]
The $m=1$ and $m=2$ branches should be treated as caveated proxy rows in one-dimensional rotation curves and as real tests only when velocity-field products are available.

\section{Why this would be a stronger Tau Core test}

The new gate is stronger than a residual association because it creates a family-selection burden. A theory with morphology-indexed readout families should not only reduce residuals; it should reduce them more when the correct morphology family is used than when a wrong family is used. MOND-simple and empirical RAR baselines are strong comparators, but they do not naturally provide a predeclared morphology-family rank test of this kind.

The result that would matter is therefore:
\[
\Delta_{\rm matched}>0
\]
with a rank distribution in which $K_{{\rm readout},g}$ is systematically near the top, under residual-blind observed morphology handles, predeclared readout-proxy rules, and shuffled-label controls. If this fails, the bridge loses specificity. If it passes, the Paper 1--3 residual-disturbance program becomes a forward-predictive morphology-readout program.

The current mixed-readout replay/holdout check is a useful negative-control
example of this burden.  Using only the fresh replay/holdout rows, NGC5907 with
its source-frozen projection mixed formula and NGC7331 with the V2
fractional-onset replay manifest both improve over Newtonian, TPG/v6, MOND,
and the exponential-disk carrier.  However, the matched labels do not beat the
wrong-label controls: the matched permutation ranks second of two and the best
shuffled assignment is slightly better by about $0.044\,{\rm km\,s^{-1}}$ in
mean RMSE.  This is preserved as a small-$N$ negative specificity result.  It
does not refute the bridge, but it shows that the current two mixed
attenuation formulas are not yet readout-specific enough to support a
replay/holdout specificity claim.

The package therefore adds a residual-blind source-side kernel-observable
separation gate, with status
\path{SOURCE\_KERNEL\_OBSERVABLE\_SEPARATION\_PASS}.  This gate does not read
observed velocities, residuals, RMSE ranks, or required-$S_\tau$ diagnostics.
It compares only source-native dimensionless features.  On the same two fresh
rows, the source fingerprints separate cleanly: NGC5907 is closer to the
projection mixed prototype than to the vertical/outer-warp V2 prototype
(\(0.921\) versus \(0.419\)), while NGC7331 is closer to the
vertical/outer-warp V2 prototype than to the projection prototype
(\(0.978\) versus \(0.177\)).  The interpretation is therefore sharper than
``the source labels are too ambiguous.''  At this stage the source observables
do separate the lanes; the bottleneck is the source-to-kernel map, where the
projection and vertical/outer-warp 4D readout kernels still become too similar
after conversion into attenuation curves.

The next preflight therefore freezes the kernel-sharpening obligation before
any new endpoint scoring.  In a dimensionless active-window coordinate \(u\),
with \(S(u)\) the smoothstep window, it proposes the source-shaped shells
\[
K_{\rm proj}^{\rm sharp}(u)
 =
S(u)^{1+\Pi_{\rm proj}+C_{\rm trunc}}
{1+C_{\rm trunc}S(u)\over 1+C_{\rm trunc}},
\]
and
\[
K_{\rm vow}^{\rm sharp}(u)
 =
S(u)e^{-(1+R_{\rm onset}/R_{\rm HI}+A_{\rm vertical})u}
 +
T_{\rm projected}S(u)(1-S(u)).
\]
The first is outer-edge/truncation dominated; the second is
onset/vertical-response dominated.  This source-side preflight has status
\path{SOURCE\_KERNEL\_SHARPENING\_PREFLIGHT\_READY\_NOT\_ENDPOINT}: it reduces
the normalized cross-similarity of the two current kernel shapes from \(0.991\)
to \(0.644\), a shape-separation gain of \(0.347\), without reading observed
velocities or residuals.  This is a formula-freeze obligation, not a fit
improvement claim: V2/V3 manifests would still need to freeze these sharper
kernels before any replay/holdout endpoint score is allowed.

That freeze-and-score check is now also executable.  The package first freezes
two source-sharpened replay formulas, with status
\path{SHARPENED\_REPLAY\_FREEZE\_READY\_NOT\_SCORED}; this freeze gate reads
only the source manifests and the sharpening preflight, and still does not
score a curve.  A separate scoring script then reads the frozen sharpened
manifest unchanged and records
\path{SHARPENED\_REPLAY\_HOLDOUT\_2CASE\_PRELIMINARY\_CONTROL\_RESULT}.  On
the same two fresh replay/holdout rows, the mean sharpened mixed RMSE is
\(19.224\,{\rm km\,s^{-1}}\), compared with \(72.919\,{\rm km\,s^{-1}}\) for
Newtonian baryonic, \(21.135\,{\rm km\,s^{-1}}\) for TPG/v6, \(23.954\,{\rm
km\,s^{-1}}\) for the MOND proxy, and \(20.421\,{\rm km\,s^{-1}}\) for the
exponential-disk carrier.  More importantly for the Tau Core-specific
question, the sharpened source-matched kernels beat all wrong-label controls in
\(2/2\) cases, and the matched diagonal is ranked first among the two shuffled
assignments.  The individual matched-vs-wrong margins are
\(-0.453\,{\rm km\,s^{-1}}\) for NGC5907 and \(-0.776\,{\rm km\,s^{-1}}\) for
NGC7331.  This does not establish population validation, but it repairs the
specific failure mode exposed by the unsharpened replay/holdout audit: after
source-native kernel sharpening, the correct morphology/readout lane is again
distinguishable from the wrong lane on the two fresh replay rows.

A final caveat-reduction audit keeps this improvement from being overstated.
It records \path{FOUR\_CASE\_CAVEAT\_REDUCTION\_AUDIT\_COMPLETE}: three of
the four inspected-case caveats are reduced, while one is isolated but not
removed.  NGC5907's prior-projection caveat is reduced to control context
because the accepted mixed gate excludes the earlier projection endpoint as
mixed evidence.  NGC7331's broad-window caveat is reduced for the replay path
by the V2 fractional onset and source-sharpened V3 kernel, but this is not a
retroactive update of the caveated accepted V1 endpoint.  NGC4088's B1
source-provenance caveat is reduced to a caveated WHISP graphical
formula-freeze input, while B2 physical normalization and B3 scale uniqueness
remain law-level caveats.  NGC4013's retrospective caveat is only isolated:
the row remains useful frozen-reference evidence, but reducing that caveat
requires a predeclared replay/holdout lane or a future source-selected analogue.
No endpoint status is changed by this audit, no endpoint score is recomputed,
and no population-validation claim is made.

Thus the present package is preparation-ready, not discovery-ready. Its next scientific upgrade is not to strengthen the claim language, but to replace available-data proxy morphology observables with accepted residual-blind morphology inputs and then run the predeclared endpoint protocol without changing gates after seeing the scores.

For that reason the repository also includes a morphology-observable intake schema. The schema records the fields required for the next run: residual-blind family labels, confidence and caveat fields, inclination and distance-quality observables, family-specific scale/support/thickness parameters, provenance, and explicit forbidden sources. In particular, it forbids using endpoint residual gain, required-$S_\tau$, endpoint-selected radii, post-hoc family choice, or per-galaxy residual tuning as morphology inputs.

A gap audit compares the proxy manifest against this intake schema. It shows that the present package is coverage-rich but acceptance-limited: active families have broad proxy coverage, while the kernel-driving observables still need accepted residual-blind morphology sources before a discovery-style endpoint can be claimed. This makes the next upgrade concrete. The task is not to alter the endpoint gates after seeing scores, but to replace proxy morphology fields by pre-scoring observables and rerun the frozen protocol.

The repository therefore records a morphology-observable source-upgrade plan. This plan prioritizes P0 family-label, confidence, caveat, and provenance fields, then P1 active-family kernel observables such as scale, tail, compact-support, and thickness parameters. Optional ring, bar, and lopsided branches remain caveated unless external morphology or velocity-field support is supplied before scoring. The plan is a source-collection protocol, not a claim that the accepted sources have already been assembled.

Finally, the repository includes an accepted-observable manifest template and validator. The template is intentionally endpoint-blocked: it keeps the galaxy rows and pre-scoring geometry fields, but leaves accepted morphology labels, kernel observables, and provenance to be supplied from residual-blind sources. This prevents the current proxy manifest from being silently promoted into a discovery input.

An accepted-manifest readiness gate then lifts the field-level validation to an endpoint-level decision. For the current empty template, the gate returns \texttt{BLOCKED\_ACCEPTED\_OBSERVABLES\_MISSING}. A future populated manifest must pass this gate before the frozen blind endpoint protocol is run.

A final frozen-endpoint launch guard records the same boundary operationally. In the current package it returns \texttt{LAUNCH\_BLOCKED} with endpoint scores not computed. Thus the package preserves a complete path to a future blind endpoint while preventing the proxy manifest or empty accepted template from being used as discovery evidence.

The package also records an external morphology source registry for the missing accepted inputs. SPARC remains the sample and baryonic/rotation baseline; S4G is the first morphology/decomposition source; NED/NED-D supplies identity, distance, and provenance checks; DustPedia is a multiband fallback/validation layer; and PHANGS is reserved for optional high-quality non-axisymmetric or velocity-field branches. A targeted \texttt{NGC4088} endpoint diagnostic is then added only as an interpretability check of the warp/history readout lane. This diagnostic is intentionally outside the source-bound proof chain: it scores against the observed rotation curve and therefore cannot be used as an accepted morphology label, a discovery endpoint, or a validation claim. Its purpose is narrower: to show what the already-generated warp readout candidates would look like when compared to the measured profile.

The diagnostic uses the residual-blind \texttt{NGC4088} readout preflight profile as its input. That profile contains the baryonic Newtonian curve \(v_N(R)\), the comparison proxy curves \(\texttt{v\_v6}(R)\) and \(\texttt{v\_mond}(R)\), the observed rotation curve \(v_{\rm obs}(R)\), and two source-generated Tau warp corrections \(\Delta v^2_{\rm p1}(R)\) and \(\Delta v^2_{\rm p2}(R)\). For each branch \(b\in\{\texttt{p1},\texttt{p2}\}\) and each predeclared sensitivity scenario \(\epsilon_{\rm cross}\), the inspected Tau curve is
\[
v_{\tau,b,\epsilon}(R)=
\sqrt{\max\!\left(v_N^2(R)
+(1+\epsilon_{\rm cross})\Delta v^2_b(R),\,0\right)} .
\]
The diagnostic then computes pointwise residuals \(v_{\tau,b,\epsilon}(R_i)-v_{\rm obs}(R_i)\) and reports RMSE, mean absolute error, bias, and inner/outer RMSE. The compared fixed baselines are \(v_N\), \(\texttt{v\_v6}\), and \(\texttt{v\_mond}\). The fixed Tau rows use \(\epsilon_{\rm cross}=0\); the bounded sensitivity rows use the previously derived locality interval for \(\epsilon_{\rm cross}\). Thus the good result below is not produced by re-fitting an arbitrary curve, but it is still an endpoint diagnostic because \(v_{\rm obs}\) is used to score and identify the best-looking bounded scenario.

Equivalently, this targeted diagnostic asks how strongly the background morphology/history layer could modulate the 4D readout in this galaxy once the leading warp candidate has been generated. The parameter \(\epsilon_{\rm cross}\) is not a new rotation-curve fit parameter; it is the bounded representative of cross-coupling between the leading warp readout and the source-side context layers, including morphology-memory, projection, and local interaction evidence. Positive \(\epsilon_{\rm cross}\) means that the background morphology layer reinforces the leading warp readout, while negative \(\epsilon_{\rm cross}\) suppresses it. The \texttt{NGC4088} result therefore tests not only whether the leading warp branch has the right shape, but also whether a source-bounded morphology-context modulation can plausibly move the 4D readout toward the observed curve.

In this diagnostic, \texttt{p1} and \texttt{p2} denote two pre-generated Tau Core warp-readout candidate branches for \texttt{NGC4088}. Both branches descend from the same source-side warp/memory/morphology construction, but they carry different first-pass readout strength. The \texttt{p1} branch is the stronger or dominant warp-readout candidate, while \texttt{p2} is the more conservative alternative. They are not two different physical laws and they are not obtained by fitting the observed residual. They are kept separately because the bridge has not yet promoted a unique branch-selection rule.

The thick orange curve in Fig.~\ref{fig:ngc4088-endpoint-diagnostic} is the bounded sensitivity case
\[
\texttt{TAU\_WARP\_p1\_eps\_+0.687500}.
\]
It is the \texttt{p1} branch evaluated at the positive edge of the previously derived source-bound \(\epsilon_{\rm cross}\) locality interval. Thus it represents the strongest currently allowed positive cross-term reinforcement of the \texttt{p1} morphology readout, not an arbitrary free fit. Because the diagnostic selects the best-looking bounded scenario after comparison with \(V_{\rm obs}\), this row remains endpoint-selected and diagnostic-only.

\[
v_{\rm proj}^2(R)
=
v_{\rm TPG}^2(R)\left[1-\gamma_{\rm proj}K_{\rm proj}(R)\right],
\qquad
\gamma_{\rm proj}
=
\frac{1}{2}\Pi_{\rm projection}\frac{h}{R}.
\]
Here \(K_{\rm proj}(R)\) is the smooth source-windowed projection kernel over the accepted warp/truncation range, and the sign is attenuation rather than added gravity because the projection-dominated lane is interpreted as an apparent overread/projection correction. With \(\Pi_{\rm projection}=0.789128\) and \(h/R=0.173321\), this gives \(\gamma_{\rm proj}=0.068386\). On the 19-point \texttt{NGC5907} curve, the diagnostic projection readout has RMSE \(15.50\,{\rm km\,s^{-1}}\), compared with \(16.79\,{\rm km\,s^{-1}}\) for the TPG/v6 proxy, \(18.60\,{\rm km\,s^{-1}}\) for the MOND proxy, \(17.03\,{\rm km\,s^{-1}}\) for the generic promoted thick/flared proxy, and \(86.48\,{\rm km\,s^{-1}}\) for the Newtonian proxy. The replacement formula shell is then derived, but not endpoint-frozen. The starting point is the solved-response convention
\[
v_{\rm readout}^2(R)=v_{\rm carrier}^2(R)+\Delta v_{\rm morph}^2(R),
\]
with \(v_{\rm carrier}=v_{\rm TPG}\) for this preflight lane. Since the source evidence points to an edge-on warp/vertical/lag overlay rather than an added-gravity tail, the residual-blind sign convention is attenuation:
\[
\Delta v_{\rm morph}^2(R)
=
-\Gamma_{\rm wvo}K_{\rm wvo}(R)v_{\rm TPG}^2(R).
\]
This gives the solved-response shell
\[
v_{\rm wvo}^2(R)
=
v_{\rm TPG}^2(R)\left[1-\Gamma_{\rm wvo}K_{\rm wvo}(R)\right].
\]
The expression is dimensionally a velocity-squared correction because
\(\Gamma_{\rm wvo}\) and \(K_{\rm wvo}\) are dimensionless. The dimensionless kernel is a source-gated mixture of the residual-blind vertical, extended-component, and lag channels:
\[
K_{\rm wvo}(R)
=
W_{\rm warp}(R;R_w,R_o)
\Big[
\omega_z K_z(R)
+\omega_{\rm EC}K_{\rm EC}(R)
+\omega_{\rm lag}K_{\rm lag}(R)
\Big],
\]
where \(W_{\rm warp}=0\) before the source warp onset and approaches \(1\) across the accepted outer warp window. The currently derived source fields give \(R_w=10\,{\rm kpc}\), \(K_z(R)=1/(1+R/R_s)\), \(K_{\rm EC}(R)=1/(1+R/z_{\rm EC})\), and \(z_{\rm EC}=3\,{\rm kpc}\). The coefficient is not fitted to the observed rotation residual. Before the lag map and weights are frozen, the allowed first-order source-side bound is
\[
\Gamma_{\rm wvo}
\le
\frac{1}{2}\frac{h}{R_s}
+
\frac{1}{2}f_{\rm EC}
=
0.216466.
\]
The limits are built into the shell: if the warp window is inactive, \(K_{\rm wvo}=0\) and \(v_{\rm wvo}=v_{\rm TPG}\); if the overlay evidence is absent, \(\Gamma_{\rm wvo}=0\) and the same carrier limit is recovered. A follow-up freeze gate then fills the three formula-level blockers at caveated protocol level. It sets \(R_o=R_{25}=11.2\,{\rm kpc}\), uses the source lag statement to define
\[
K_{\rm lag}(R)
=
\mathrm{clip}
\left(
\frac{R_{25}-R}{R_{25}-R_{\rm lag,start}},
0,1
\right),
\qquad
R_{\rm lag,start}=5.8\,{\rm kpc},
\]
and freezes the temporary residual-blind weight rule
\(\omega_z=\omega_{\rm EC}=\omega_{\rm lag}=1/3\). This resolves the formula-freeze blockers without using \(v_{\rm obs}\) or endpoint residuals. It still does not permit endpoint scoring: the compact lane has been rejected, and the replacement \path{K\_warp\_vertical\_overlay\_candidate} label has not yet been promoted by an accepted morphology manifest. The current status is therefore \path{FORMULA\_FREEZE\_PROTOCOL\_READY\_LABEL\_BLOCKED}, not an accepted endpoint.

The replacement-label gate then performs that source-side promotion step. It combines the negative compact evidence, the warp/overlay pressure field, the line-of-sight warp onset, the S4G edge-disk \(h/R\), the extended vertical component, the central H I scaleheight, and the rotational-lag context. All six promotion gates pass, with the formula-freeze gate marked caveated rather than unconditional because \(R_o=R_{25}\), the lag kernel is a linear shell, and the weights are uniform until a stronger source-evidence hierarchy is derived. The resulting status is \path{CAVEATED\_REPLACEMENT\_LABEL\_PROMOTED\_ENDPOINT\_SCORE\_ALLOWED}. This means that a preliminary NGC4013 endpoint may now be run with the frozen warp/vertical-overlay formula, while the claim remains a caveated single-galaxy control and not a population validation.

\section{Conclusion}

Paper 3 is the natural launching point for this forward gate because it already contains the candidate/control framework and the required-$S_\tau$ diagnostic. The present contribution is to state the next gate in a falsifiable form:
\[
\begin{aligned}
&\text{observed morphology handle }K_{\rm obs}\\
&\quad\to\text{ readout proxy }K_{\rm readout}\\
&\quad\to\text{ formula shell }\mathcal{F}_{K_{\rm readout}}\\
&\quad\to \delta g^{K_{\rm readout}}
\to\text{ matched vs wrong family endpoint}.
\end{aligned}
\]

The claim-safe breakthrough target is not a universal fit claim. It is a morphology-specific prediction claim: the correct Tau Core morphology readout family, chosen before endpoint scoring, should improve residual structure more than wrong morphology families and shuffled morphology labels, while remaining competitive against Newtonian baryonic, MOND-simple, and empirical RAR baselines.

The executable package has now reached a sharper preliminary status than the original proposal: it contains four inspected readout cases, split into a three-case mixed-readout control packet, accepted \texttt{NGC5907} and caveated \texttt{NGC7331} mixed endpoint promotions inside that packet, and one additional caveated \texttt{NGC4088} accepted endpoint. In all four inspected rows, the matched Tau Core readout beats the best local baseline and the inspected wrong-family controls. This is not yet population validation, but it is the first compact evidence packet for the paper's intended claim: source- or morphology-matched readout structure can matter in a way that generic baselines and wrong readout families do not automatically reproduce.

The NGC4013 mixed-readout lane illustrates the intended standard. The paper now separates three statements that should not be conflated: source evidence supports a mixed smooth-disk plus warp/vertical-overlay interpretation; the approximate mixed 4D readout formula can be frozen residual-blind as \(v_{\rm mix}^2=v_{\rm expdisk}^2(1-\Gamma_{\rm wvo}K_{\rm wvo})\); and the already inspected numerical score is not a retroactive accepted endpoint. The new blocker gate makes that last point executable rather than rhetorical: \texttt{NGC4013} is protocol-ready for future use, but accepted endpoint status requires a predeclared replay/holdout lane or a future source-selected case. This is the kind of disciplined intermediate result the program needs before population-level endpoint claims.

That is the first clean endpoint where the Tau Core bridge can become more than residual commentary.
